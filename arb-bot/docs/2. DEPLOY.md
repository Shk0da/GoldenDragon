# 🚀 Краткая инструкция: деплой в Ethereum Mainnet

## 📋 Что нужно подготовить

1. **MetaMask** установлен и настроен на **Ethereum Mainnet**
2. **Два кошелька** в MetaMask:
    - `Arb Owner` — деплоер (минимум **0.1 ETH** для газа)
    - `Arb Profit` — для приёма прибыли
3. **Адреса сохранены локально** в текстовый файл
4. **Remix IDE** открыт: [remix.ethereum.org](https://remix.ethereum.org)

---

## 🛠 Шаг 1. Создай файл в Remix

В Remix создай файл `HonestFlashArbV2.sol` и вставь код контракта (финальная версия с автовыводом из предыдущих сообщений).

---

## ⚙️ Шаг 2. Компиляция

Вкладка **Solidity Compiler**:
- Compiler version: `0.8.21` или новее
- ✅ Enable optimization: `200`
- Нажми **Compile HonestFlashArbV2.sol**
- Дождись зелёной галочки

---

## 🔌 Шаг 3. Подключи MetaMask

Вкладка **Deploy & Run Transactions**:
- **Environment**: `Injected Provider - MetaMask`
- В MetaMask выбран аккаунт **Arb Owner**
- Сеть: **Ethereum Mainnet (1)**

---

### TOKENS

**Расшифровка:**

- Tokens = WETH, USDC, USDT, DAI, WBTC

---

## 🚀 Шаг 5. Деплой

1. Нажми оранжевую кнопку **transact**
2. В MetaMask проверь:
    - Сеть: **Ethereum Mainnet**
    - Аккаунт: **Arb Owner**
    - Газ: разумный (используй [etherscan.io/gastracker](https://etherscan.io/gastracker))
3. Подтверди транзакцию
4. Дождись 15-60 секунд
5. **Скопируй адрес задеплоенного контракта** из раздела `Deployed Contracts`

---

## ⚙️ Шаг 6. Настройка после деплоя

В Remix в разделе `Deployed Contracts` найди твой контракт и вызови по очереди:

### 6.1. Установить получателя прибыли

Функция `setProfitReceiver` → вставь адрес `Arb Profit`:

```
0xТВОЙ_АДРЕС_ARB_PROFIT_ЗДЕСЬ
```

→ **transact** → подтверди в MetaMask

### 6.2. Установить пороги автовывода (100 единиц для каждого токена)

Используй `setAutoWithdrawThresholds` (множественная установка):

**Поле `tokens`:**

```
["0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48","0xdAC17F958D2ee523a2206206994597C13D831ec7","0x6B175474E89094C44Da98b954EedeAC495271d0F","0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2","0x2260FAC5E5542a773Aa44fBCfeDf7C193bc2C599"]
```

**Поле `thresholds`:**

```
["100000000","100000000","100000000000000000000","100000000000000000","10000000"]
```

→ **transact** → подтверди

**Расшифровка порогов:**
- USDC: `100000000` = 100 USDC (6 decimals)
- USDT: `100000000` = 100 USDT (6 decimals)
- DAI: `100000000000000000000` = 100 DAI (18 decimals)
- WETH: `100000000000000000` = 0.1 WETH (18 decimals)
- WBTC: `10000000` = 0.1 WBTC (8 decimals)

## 🎯 Шаг 8. Запуск арбитража

Когда найдёшь арбитражную возможность, вызови `startArbitrage` с параметрами:

### asset

Адрес токена, который будешь занимать (например, USDC):

```
0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48
```

### amount

Сумма займа в наименьших единицах (например, 10 000 USDC = 10000 * 10^6):

```
10000000000
```

### plan (структура — в Remix будут раскрытые поля)

**router1** (где покупаешь, например Uniswap):
```
0x7a250d5630B4cF539739dF2C5dAcb4c659F2488D
```

**router2** (где продаёшь, например SushiSwap):
```
0xd9e1cE17f2641f24aE83637ab66a2cca9C378B9F
```

**path1** (USDC → WETH):
```
["0x94a9D9AC8a22534E3FaCa9F4e7F2E2cf85d5E4C8","0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2"]
```

**path2** (WETH → USDC):
```
["0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2","0x94a9D9AC8a22534E3FaCa9F4e7F2E2cf85d5E4C8"]
```

**amountOutMin1** (минимум WETH за 10k USDC, рассчитай через API роутера, с slippage 0.5%):
```
[твой_расчёт]
```

**amountOutMin2** (минимум USDC обратно — должно быть больше суммы займа + комиссия Aave 0.05%):
```
10010000000
```

**minProfit** (минимальная чистая прибыль, например 10 USDC):
```
10000000
```

**deadline** (Unix timestamp = текущее время + 60 секунд, получи на [unixtimestamp.com](https://www.unixtimestamp.com)):
```
1733980000
```

→ **transact** → подтверди → дождись результата.

## 💰 Как забрать накопленную прибыль вручную (если порог не достигнут)

Функция `withdrawAccumulatedProfit` → введи адрес токена:

```
0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48
```

→ **transact** → подтверди. Прибыль уйдёт на `profitReceiver`.

## 🚨 Аварийная остановка и эвакуация средств

### Поставить на паузу

Функция `pause` → **transact**

### Снять с паузы

Функция `unpause` → **transact**

### Вывести любые токены (только когда на паузе)

Функция `sweepToken` с параметрами:

```
token:  0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48
to:     0xТВОЙ_АДРЕС_КУДА_ВЫВЕСТИ
amount: 1000000000
```

→ **transact**

## 📊 Верификация контракта на Etherscan (опционально, но рекомендуется)

После деплоя верифицируй код, чтобы он был публично читаем:

1. Открой [etherscan.io](https://etherscan.io) → найди свой контракт по адресу
2. Вкладка **Contract** → **Verify and Publish**
3. Compiler Type: `Solidity (Single file)`
4. Compiler Version: тот же, что в Remix
5. License: `MIT`
6. Вставь код контракта
7. Добавь Constructor Arguments в ABI-encoded виде:
    - Скопируй из Remix → в логах деплоя есть `input` транзакции
    - Или используй [abi.hashex.org](https://abi.hashex.org) для энкодинга
8. **Verify and Publish**

## ⚠️ Финальные предупреждения

- **Не используй основной кошелёк** как Arb Owner
- **Тестируй сначала с малыми суммами** (например, заём 100 USDC, а не 100k)
- **Следи за газом** — если он зашкаливает, арбитраж невыгоден
- **MEV-боты могут опережать тебя** — рассмотри Flashbots для приватной отправки
- **Адреса роутеров/токенов проверь дважды** перед деплоем
